memory, knowledge, and search are what make float feel persistent instead of forgetful. This part of the system stores facts, notes, uploaded material, and other useful context so float can bring the right information back later instead of starting from zero in every conversation.

From a user's point of view, this means float can remember a preference, keep a project note, search a saved document, or show which stored snippets influenced a reply. The important distinction is that memory is the specific thing float should keep, knowledge is the wider library of saved material, and retrieval is the step where float tries to find the most relevant piece for the current moment.

This area is powerful but still imperfect because retrieval is a ranking system, not magic recall, and some stored knowledge types are more mature than others. The public references for the current shipped surface are `docs/api_reference.md` and `docs/data_directory.md`.
