calendar, tasks, and follow-ups are the part of float that turns a conversation into something scheduled, actionable, and reviewable later. Instead of keeping everything inside the current chat, float can store an event, a reminder, or a task that carries actions forward into the future.

What makes this more than a normal calendar is that a scheduled item can still behave like part of the assistant. A follow-up can point back to a chat, carry a structured action list, or return later as a prompt that asks the user for review. That makes this feature less about date storage alone and more about helping work actually continue.

This system is increasingly unified, but it is still being refined, especially where tasks, events, and automated follow-up behavior overlap. The public endpoint reference for the current shipped surface is `docs/api_reference.md`.
