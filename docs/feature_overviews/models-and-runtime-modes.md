For the user, this is really about tradeoffs. One mode might be better for privacy, another for speed, another for capability, and another for hardware convenience. The important distinction is that local means the work is happening on the user's machine or through a locally managed runtime, while Server/LAN means float is talking to some other server that already exists.

In practice there are three separate categories to keep straight: direct local Transformers checkpoints running inside Float, managed local providers such as LM Studio or Ollama that Float talks to over an OpenAI-compatible interface, and `Server/LAN` targets that are already-running remote or networked servers. Those paths overlap in the UI, but they are not identical in maturity or configuration. This flexibility is one of float's core design goals, but feature parity across all runtimes is still a work in progress and some runtime paths need more live verification than others.
