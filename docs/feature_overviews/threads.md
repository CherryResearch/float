threads is float's way of grouping separate conversations into larger topic bundles. Instead of making a person browse chat history one file at a time, it tries to surface broader themes so recurring ideas, projects, or subjects are easier to explore.

For the user, this works like a higher-level map of conversation history. A thread can show a topic card, related snippets, linked conversations, and sub-threads that break a broad subject into smaller branches. The point is not to replace normal chats, but to make a growing history feel structured and explorable.

This feature is intentionally manual and on-demand because float should not silently reorganize everything without the user choosing to refresh it. The public references for the current shipped surface are `docs/api_reference.md` and `docs/architecture_map.md`.
