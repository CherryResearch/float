the agent console is where float shows active agents and subagents instead of hiding that work behind a single reply. If the main chat kicks off a larger task, the console is the place where delegated work can stay visible as separate threads of activity rather than disappearing into the background.

For the user, this should feel like oversight instead of interruption. A person can glance at active agent cards, see what each subagent is working on, open the related sub-chat, and decide whether to let the work continue, step in, or redirect it. The main chat stays the orchestrator, while the console keeps parallel work readable.

This matters because float is built to branch work when needed instead of forcing everything through one opaque answer stream. The public implementation-facing references in this snapshot are `docs/architecture_map.md` and `docs/api_reference.md`.
